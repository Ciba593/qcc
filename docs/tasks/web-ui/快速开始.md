# QCC Web UI - 快速开始指南

## 🚀 安装

### 方法 1: 使用安装脚本（推荐）

```bash
# 进入项目目录
cd qcc

# 运行安装脚本
./setup-web.sh
```

安装脚本会自动：
- 检查 Python 环境
- 安装 QCC 及所有 Web 依赖
- 完成后提示启动命令

### 方法 2: 手动安装

```bash
# 安装 QCC 及 Web 依赖
pip install -e '.[web]'
```

## 🎯 启动 Web UI

### 基本启动

```bash
# 启动 Web UI（默认端口 8080）
qcc web start

# 指定端口
qcc web start --port 9000

# 指定主机地址
qcc web start --host 0.0.0.0 --port 8080
```

### 开发模式

```bash
# 开发模式（自动重载）
qcc web start --dev

# 开发模式 + 不自动打开浏览器
qcc web start --dev --no-browser
```

### 停止服务

```bash
# 停止 Web 服务器
qcc web stop
```

## 🌐 访问 Web UI

启动成功后，在浏览器中访问：

- **Web UI**: http://127.0.0.1:8080
- **API 文档（Swagger）**: http://127.0.0.1:8080/api/docs
- **API 文档（ReDoc）**: http://127.0.0.1:8080/api/redoc

## 📖 主要功能

### 1. 仪表盘（Dashboard）

查看系统概览：
- 配置总数和存储方式
- 代理服务状态
- Endpoint 健康状况
- 代理组管理

### 2. 配置管理（Configs）

- ✅ 查看所有配置
- ✅ 创建新配置
- ✅ 编辑配置
- ✅ 删除配置
- ✅ 使用配置（应用到 Claude Code）
- ✅ 设置默认配置
- ✅ 同步云端配置

### 3. Endpoint 管理

- ✅ 创建 EndpointGroup（高可用代理组）
- ✅ 管理主节点和副节点
- ✅ 删除代理组

### 4. 代理服务（Proxy）

- ✅ 启动代理服务
- ✅ 停止代理服务
- ✅ 查看代理日志
- ✅ 实时监控代理状态

### 5. 健康监控（Health）

- ✅ 查看所有节点健康状态
- ✅ 成功率和响应时间统计
- ✅ 运行时节点管理（添加/移除/移动）
- ✅ 失败队列查看和重试

## 🔧 开发模式

如果你要开发前端，可以使用前端开发服务器（支持热模块替换）：

```bash
# 1. 启动后端开发服务器
qcc web start --dev --no-browser

# 2. 在另一个终端启动前端开发服务器
cd qcc-web
npm run dev
```

前端开发服务器地址：http://localhost:5173

## 🐛 常见问题

### 端口被占用

```bash
# 查看占用 8080 端口的进程
lsof -i :8080

# 使用其他端口
qcc web start --port 9000
```

### 前端页面空白

```bash
# 重新构建前端
cd qcc-web
npm run build
cp -r dist/* ../fastcc/web/static/

# 重启服务
cd ..
qcc web stop
qcc web start
```

### 查看日志

```bash
# Web 服务日志
tail -f ~/.fastcc/web.log

# 代理服务日志
qcc proxy logs
```
